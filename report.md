# Project Report: FocusSense – Study Behavior Analyzer

## 1. Introduction

This project focuses on analyzing student study behavior rather than just tracking time. The idea is to move from raw input to meaningful insight.

## 2. Problem Statement

Students often spend long hours studying but fail to measure how much of that time is actually productive. Distractions reduce efficiency but are rarely quantified.

## 3. Objective

To create a simple system that:
- Captures study and distraction time
- Evaluates productivity
- Identifies weak patterns
- Provides actionable feedback

## 4. Approach

The project collects multiple sessions of input from the user.

Each session includes:
- Study time
- Distraction time

The system then:
- Calculates total productivity
- Finds the lowest performing session
- Generates feedback based on performance levels

## 5. Implementation

The solution is implemented in Python using:
- Lists for storing session data
- Functions for modular logic
- Loops for iteration
- Conditional logic for decision-making

## 6. Key Features

- Multi-session tracking
- Productivity calculation
- Weakest session detection
- Behavior-based feedback

## 7. Challenges Faced

- Handling invalid user inputs
- Designing meaningful productivity thresholds
- Structuring the flow for multiple sessions

## 8. Learning Outcomes

- Improved understanding of user behavior modeling
- Practical use of loops and conditions
- Translating a real-world issue into a logical solution

## 9. Conclusion

FocusSense demonstrates how simple computational logic can be used to generate insights about behavior. It highlights that even small tools can create awareness and drive improvement.
